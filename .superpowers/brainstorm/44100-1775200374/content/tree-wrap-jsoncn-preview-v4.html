续行顶左侧效果

这版按你的意思处理：第一行仍然是 key: value，但一旦 value 折行，后续续行直接顶着当前节点内容区的最左侧开始，不再保留 key 的缩进占位。

Format 输出预览（续行顶左）
▼ [
▼ 0 : {
"id": null,
"stepType": 0,
"stepContent": "{\"apiId\":10356,\"apiName\":\"获取提报预测周期总览列表\",
\"envId\":60,\"authId\":31,\"apiPath\":\"/budget/api/predict/cycle/manage/list\",
\"queryParam\":{\"pageSize\":\"50\",\"testUser\":\"${test_user}\",\"pageNum\":\"1\"},
\"variableList\":[],\"preRequestScript\":\"\",\"aiPreRequestScript\":false}"
"stepName": "0. 获取列表并找到预算测试周期ID",
"createAt": "2026-04-03 14:17:52",
"createdBy": "wb_shijintao"
}

这版变化

  • 第一行仍保留 `key: value` 关系
  • 续行直接顶着当前层级的最左起点
  • 不会继续占用 key 的水平空间

实现思路

  • 长字符串拆成“首行 + 续行”渲染
  • 续行单独输出为同层级的 value 行
  • 只影响长字符串 primitive 节点
