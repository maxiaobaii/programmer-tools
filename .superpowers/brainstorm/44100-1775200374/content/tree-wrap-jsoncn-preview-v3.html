自然换行（整行流式）效果

这次我改成和你截图更接近的表现：整行作为一段文本流，不是把 value 放在独立列里。所以续行会回到当前节点起点，而不是继续挂在 value 列下面。

Format 输出预览（整行自然换行）
▼ [
▼ 0 : {
"id": null,
"stepType": 0,
"stepContent": "{\"apiId\":10356,\"apiName\":\"获取提报预测周期总览列表\",\"envId\":60,\"authId\":31,\"apiPath\":\"/budget/api/predict/cycle/manage/list\",\"queryParam\":{\"pageSize\":\"50\",\"testUser\":\"${test_user}\",\"pageNum\":\"1\"},\"variableList\":[],\"preRequestScript\":\"\",\"aiPreRequestScript\":false}"
"stepName": "0. 获取列表并找到预算测试周期ID",
"createAt": "2026-04-03 14:17:52",
"createdBy": "wb_shijintao"
}

这次和上一版的关键区别

  • 上一版：`key / colon / value` 是分列布局，续行还在 value 列里
  • 这版：整行 inline 流式布局，续行回到当前节点内容起点
  • 因此不会出现第二行从很右边继续挂着的感觉

实现方式

  • primitive 节点改成单个可换行文本容器
  • key、冒号、value 仍分别着色，但不再拆成 value 专属列
