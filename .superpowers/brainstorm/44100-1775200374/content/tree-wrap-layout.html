Tree value 换行方式

A 是当前效果，B 是推荐效果。重点看长字符串 value 的换行起点是否影响 key 对齐。

A. 当前效果
stepContent : "{"apiId":10356,"apiName":"获取提报预测周期总览列表","envId":60,"authId":31,"apiPath":"/budget/api/predict/cycle/..."
"pathParam":{},"queryParam":{"pageSize":"50",...}
"variableList":[],"preRequestScript":"" ..."
stepName : "0. 获取列表并找到预算测试周期ID"
createAt : "2026-04-03 14:17:52"
B. 推荐效果
stepContent :
"{"apiId":10356,"apiName":"获取提报预测周期总览列表","envId":60,"authId":31,"apiPath":"/budget/api/predict/cycle/..."
"pathParam":{},"queryParam":{"pageSize":"50",...}
"variableList":[],"preRequestScript":"" ..."
stepName : "0. 获取列表并找到预算测试周期ID"
createAt : "2026-04-03 14:17:52"

推荐 B 的原因

  • 后续 key 行会重新从统一列开始，对齐更稳定
  • 长 value 不会把视觉焦点向右推得太远
  • 实现简单，只改 `TreeNode` 的 primitive 行布局

代价

  • 长 value 会比现在占更多垂直空间
  • 短 value 也可能需要保持原行内展示，避免整体过高
