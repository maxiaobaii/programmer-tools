# JSON Pro 移动端适配设计

- 日期：2026-03-31
- 主题：在不改变桌面端逻辑的前提下，适配手机与 pad 端

## 1. 目标

本轮只做响应式与触屏交互增强，不改变桌面端的信息架构与功能逻辑。

目标：

1. 手机端可正常完成输入、查看结果、切换工具、复制/下载等核心操作
2. pad 端横竖屏都具备更合理的布局密度
3. 桌面端布局逻辑、交互流程、功能输出保持不变

## 2. 设计原则

- **桌面端不回退**：桌面端仍保持当前顶部导航 + 主区域双栏/多栏结构
- **手机端强移动优先**：双栏页默认纵向堆叠，优先保证输入与结果可读性
- **pad 端折中**：横屏尽量保留双栏，竖屏允许切换为纵向布局
- **只改展示，不改核心功能**：不改变格式化、校验、Diff、转换等已有逻辑

## 3. 断点策略

- `<= 767px`：手机
- `768px - 1180px`：pad / 小屏平板
- `> 1180px`：桌面

附加判断：
- pad 竖屏：更偏向纵向堆叠
- pad 横屏：可保留部分双栏

## 4. 布局策略

### 4.1 App 与导航
- 桌面 / pad 横屏：保留顶部 tab 导航
- 手机：顶部仅保留品牌与主题按钮，工具切换移到**底部 tab bar**
- 底部 tab bar 使用现有 tab 数据，不改变切换逻辑

### 4.2 ResizableLayout
- 桌面端：继续保留拖拽分栏
- pad 竖屏 / 手机：自动切换为纵向堆叠，隐藏拖拽分割条

### 4.3 页面区块
- `Format / Compress / Convert / Types / JsonPath / Tree / Escape`
  - 手机端上下堆叠
- `Validate`
  - 手机端形成 JSON → Schema → Result 的纵向流式阅读
- `Diff`
  - 手机端输入 A / B / 结果顺序排列
  - 差异卡片继续使用更紧凑布局

## 5. 交互策略

- 增大移动端按钮高度和点击区域
- 操作按钮允许换行，不压缩到不可点击
- 主区域为手机端预留底部 tab bar 安全空间
- 输入框、结果区最小高度增大，避免过短导致难编辑

## 6. 代码结构

新增：
- `src/lib/viewport.js`：断点与方向判断纯函数
- `src/composables/useViewport.js`：监听窗口变化并输出响应式视口状态

修改：
- `src/App.vue`
- `src/components/ResizableLayout.vue`
- `src/style.css`

## 7. 验证标准

完成后应满足：

- 桌面端视觉与交互逻辑保持现状
- 手机端出现底部 tab bar，主区域不被遮挡
- 手机端所有双栏工具页改为更易读的纵向布局
- pad 竖屏能正常堆叠，横屏保持较高信息密度
- `npm test`、`npm run lint`、`npm run type-check`、`npm run build` 通过
